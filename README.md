# TerraFlow Ops

Internal operations console for [TerraFlow.Studio](https://terraflow.studio). Replaces the 10-tab Excel tracker with a multi-user web app: tasks, clients, pipeline, revenue, cash flow, cold-email tracking.

Production at `app.terraflow.studio` once Vercel is connected.

## Stack

- **Next.js 16** · App Router, RSC by default, Turbopack, TypeScript strict
- **Tailwind v4** · brand tokens declared via `@theme` in `src/app/globals.css`
- **Supabase** · Postgres, Auth (magic-link), RLS, Realtime; `@supabase/ssr` for Next.js integration
- **shadcn/ui** · 20 primitives in `src/components/ui/`, dark-only palette
- **TanStack Query v5** · client-side cache; provider in `src/components/providers.tsx`
- **react-hook-form + zod** · all forms
- **Lucide** · only icon library
- **pnpm** · package manager

## Authentication model

This is a closed app — only the 5 named team members can sign in. Auth uses **email + password** (no magic link, no open signup, no third-party providers).

To enforce this, in the Supabase dashboard:

1. **Authentication → Providers → Email** — keep "Enable email provider" ON, but turn **OFF** "Enable email signups" (and any "Confirm email" requirement). This means no one can self-create an account; only admin-created users exist.
2. **Authentication → Providers** — disable any other providers you don't use (GitHub, Google, etc.) for tidiness.

## Windows + OneDrive caveat

This project lives inside a OneDrive-synced folder. To prevent OneDrive from corrupting `node_modules` and `.next`, both directories are **junction-mounted** to `C:\dev\terraflow-cache\` so the heavy files live outside OneDrive. The junctions were created during initial setup. If you re-clone or move the project:

```powershell
$proj = "C:\path\to\Terraflow.app"
$cache = "C:\dev\terraflow-cache"
New-Item -ItemType Directory -Force -Path "$cache\node_modules", "$cache\.next" | Out-Null
cmd /c "rmdir /s /q `"$proj\node_modules`""
cmd /c "rmdir /s /q `"$proj\.next`""
cmd /c "mklink /J `"$proj\node_modules`" `"$cache\node_modules`""
cmd /c "mklink /J `"$proj\.next`" `"$cache\.next`""
```

Long-term recommendation: move the whole repo to `C:\dev\terraflow-app\` and skip OneDrive entirely. The junctions are a workaround.

## Deployment

Vercel will build and host. The repo is connected via the Vercel dashboard (no CLI needed).

### First deploy

1. Go to <https://vercel.com/new> and "Import Git Repository" → select `Obitus599/terraflow.app`.
2. **Framework preset:** Next.js (auto-detected).
3. **Root directory:** `./` (default).
4. **Build command:** `next build` (default). On Vercel, Turbopack works fine — no need to override.
5. **Environment variables** — paste these three (production scope):

   ```env
   NEXT_PUBLIC_SUPABASE_URL=https://ocnaoeaiqebtoizbjlni.supabase.co
   NEXT_PUBLIC_SUPABASE_ANON_KEY=<anon key from Supabase, Settings, API>
   SUPABASE_SERVICE_ROLE_KEY=<service_role key from same page>
   ```

   Do NOT add `SUPABASE_ACCESS_TOKEN` or `SUPABASE_DB_PASSWORD` — those are CLI-only and don't belong in runtime env.

6. Click **Deploy**. First deploy takes ~3 minutes.
7. Note the assigned URL (e.g. `terraflow-app-xyz.vercel.app`).

### Custom domain + branch protection

1. **Custom domain (optional, recommended):** in Vercel project → Settings → Domains → add `app.terraflow.studio`. Then in GoDaddy DNS, add a CNAME: `app` → `cname.vercel-dns.com`. Vercel auto-issues the SSL cert.
2. **Branch protection:** in GitHub repo → Settings → Branches → add a rule for `main` that requires PRs and passing checks.

No Supabase URL / redirect URL configuration is needed — password auth doesn't use email callbacks.
